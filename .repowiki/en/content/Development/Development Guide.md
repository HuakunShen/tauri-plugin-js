## Project Setup

### Prerequisites

- **Rust** — Install via [rustup](https://rustup.rs/) (version 1.77.2+)
- **pnpm** — Package manager for JavaScript
- **Tauri v2 CLI** — `cargo install tauri-cli`

### Clone and Install

```bash
git clone https://github.com/HuakunShen/tauri-plugin-js.git
cd tauri-plugin-js

# Install JS dependencies and build guest-js
pnpm install
pnpm build
```

### Build Script

The `build.rs` script:

1. Defines the command list for permission generation
2. Embeds the target triple for sidecar resolution

```rust
const COMMANDS: &[&str] = &[
    "spawn",
    "kill",
    "kill_all",
    "restart",
    "list_processes",
    "get_status",
    "write_stdin",
    "detect_runtimes",
    "set_runtime_path",
    "get_runtime_paths",
];

fn main() {
    println!(
        "cargo:rustc-env=TARGET_TRIPLE={}",
        std::env::var("TARGET").unwrap()
    );

    tauri_plugin::Builder::new(COMMANDS)
        .android_path("android")
        .ios_path("ios")
        .build();
}
```

## Permissions

### Default Permissions

The `permissions/default.toml` defines the default permission set:

```toml
[default]
description = "Default permissions for the plugin"
permissions = [
    "allow-spawn",
    "allow-kill",
    "allow-kill-all",
    "allow-restart",
    "allow-list-processes",
    "allow-get-status",
    "allow-write-stdin",
    "allow-detect-runtimes",
    "allow-set-runtime-path",
    "allow-get-runtime-paths",
]
```

**Section sources**

- [permissions/default.toml](file://permissions/default.toml)

### Auto-Generated Permissions

The `tauri-plugin` build macro generates per-command permissions in `permissions/autogenerated/commands/`:

- `spawn.toml` — `allow-spawn`, `deny-spawn`
- `kill.toml` — `allow-kill`, `deny-kill`
- etc.

### Capability Configuration

In the consuming app, add to `src-tauri/capabilities/default.json`:

```json
{
  "permissions": [
    "core:default",
    "js:default"
  ]
}
```

## Code Style

### Rust

- **Error handling**: Use `thiserror` for custom errors in `error.rs`
- **Async**: Use `tokio` for async operations
- **State**: Use `Arc<Mutex<T>>` pattern for shared state
- **Naming**: Snake_case for functions/variables
- **Events**: camelCase for event payloads (via `#[serde(rename_all = "camelCase")]`)

**Section sources**

- [src/error.rs](file://src/error.rs)
- [src/models.rs](file://src/models.rs)

### TypeScript

- **Types**: Explicit interfaces for all public APIs
- **Imports**: Use ESM imports
- **Naming**: camelCase for functions/variables, PascalCase for types/classes
- **Async**: Always use async/await, not raw promises

**Section sources**

- [guest-js/index.ts](file://guest-js/index.ts)
- [tsconfig.json](file://tsconfig.json)

### Anti-Patterns to Avoid

1. **Do not parse RPC in Rust** — Rust is a thin relay
2. **Do not forget `\n`** — Frontend must re-append newline
3. **Do not hold mutex across await** — Take data out of lock before `.await`
4. **Do not drop stdin before taking** — Use `entry.stdin.take()` before `child.kill()`
5. **Do not block kkrpc read() with null** — Return never-resolving promise when destroyed
